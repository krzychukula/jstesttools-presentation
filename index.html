describe('How about async?', function() {
  it('should calc asynchronously', function () {
    var spreadsheet = new Spreadsheet();
    spreadsheet.asynchronouslyCalculate();

    waitsFor(function() {
      return spreadsheet.calculationIsComplete();
    }, "Spreadsheet calculation never completed", 10000);

    runs(function () {
      expect(spreadsheet.total).toEqual(123456);
    });
});});

Node.js alternative to JsTestDriver
  • Conf is in JS file
  • BDD and Unit syntax
  • Static html tests
  • Many reporters eg. xml, dots
  • Not tested, but looks better than JTD
Mocha
  • UnitTest library
  • available via npm
  • BDD syntax
  • Many reporters

  • Headless WebKi
  • DOM handling, CSS selector, JSON, Canvas, and SVG.
  • Think of it like command line browser :)
  • I want't able to install it :(

jsdom, Zombie

  • Node.js libraries simulating window and DOM
  • Can used for testing
  • Easy install via npm
  • Can have yet another bugs than browsers
  • creates it's own window, can get external sites

Grunt, node-build-script

  • Linting, testing
  • Run on localhost (dev, prod versions)
  • Easy install via npm, git clone
  • Rather beta, but can bu used
  • ant-build-script from h5bp is more stable
